# E-Commerce-API-Integration-BA-Project


## Project Overview
This project demonstrates the end-to-end responsibilities of a Business Analyst in designing an API-integrated e-commerce order processing system.

## Business Problem
Manual coordination between e-commerce systems, payment gateways, and vendors can lead to delays, errors, and poor customer experience.

## Solution
The project proposes an automated system using APIs to:
- Check product availability
- Process payments
- Confirm orders
- Handle shipment updates
- Notify customers

## My Role
- Created BRD and FRD
- Designed AS-IS and TO-BE process flows (BPMN)
- Defined business rules
- Prepared UAT test cases
- Analysed API success and failure scenarios
